Data Size

The dataset generated contains from 10 to 100 character per record, as we have 100 record (column), per row and 6000 row which makes about 60,000 Record,the size of the data is huge, which will overload the main thread when the app is initializing which in worse case can lead to the browser/tab to crash and provide terrible experience.

Web Worker

The generation of the dataset needs to be moved from the main thread, and that is possible with webworker. Web worker in a thread asyncly communicating with the main thread which is responsible for rendering the interface, the CPU intensive tasks as generating 60,000 records can run in separte thread and then destoryed to prevent memory leak.
