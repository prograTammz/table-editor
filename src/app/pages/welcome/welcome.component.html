Data Size

The dataset generated contains from 10 to 100 character per record, as we have 100 record (column), per row and 6000 row which makes about 60,000 Record,the size of the data is huge, which will overload the main thread when the app is initializing which in worse case can lead to the browser/tab to crash and provide terrible experience.

Web Worker

The generation of the dataset needs to be moved from the main thread, and that is possible with webworker. Web worker in a thread asyncly communicating with the main thread which is responsible for rendering the interface, the CPU intensive tasks as generating 60,000 records can run in separte thread and then destoryed to prevent memory leak.

Data Storage üíæ

Local Storage

The dataset has60,000 Record,the size of the data is huge, about 10MB in size. The first logical storage method is Local Storage which needs the data to be parsed to String via JSON.stringify() at which the file can go nearly 35MB, and that's above the allowed quota for local storage

IndexedDB

The indexedDB is made for that purpose, it allow you to make a NO-SQL database and perform queries like MongoDB which also made for high performance queries, it runs in a different thread different from the main thread, which won't negatively impact the performance.

Dexie.js

Dexie.js is used as it provides indexedDB promise wrapper and cool functionalities and extremely small in size, as well as being well maintained
